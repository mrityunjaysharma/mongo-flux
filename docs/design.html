1. Architecture

MongoDB runs as a 3-node replica set (rs0): 1 primary + 2 secondaries. For reads, the application connects to MongoFlux which acts as a routing proxy. If the connection URI contains ?clickhouse=true, MongoFlux translates the query to SQL and sends it to ClickHouse. If the parameter is absent or false, MongoFlux forwards the query to MongoDB Primary unchanged. Writes go directly to the primary.

2. Data Flow

Write Path (zero overhead)

Writes target the MongoDB primary. The two secondaries replicate via the standard MongoDB protocol. MongoFlux tails the primary's oplog independently.

sequenceDiagram participant App participant Primary as MongoDB Primary participant S1 as Secondary 1 participant S2 as Secondary 2 participant mf as MongoFlux participant CH as ClickHouse App->>Primary: insert/update Primary-->>App: ack ✓ Primary->>S1: replication (async) Primary->>S2: replication (async) Note right of App: Write done.
MongoFlux not in path. Primary->>mf: oplog entry (async) mf->>CH: batch INSERT mf->>mf: save position

Read Path (query routing)

The application sends all reads to MongoFlux. If ?clickhouse=true is in the URI, the query is translated to SQL and executed on ClickHouse. Otherwise, it's forwarded to MongoDB Primary unchanged.

sequenceDiagram App->>MongoFlux: find({status:"shipped"}, ?clickhouse=true) MongoFlux->>MongoFlux: BSON → AST → SQL MongoFlux->>ClickHouse: SELECT * WHERE status='shipped' ClickHouse-->>App: results
sequenceDiagram App->>MongoFlux: find({_id: "abc123"}) [no clickhouse param] MongoFlux->>MongoDB Primary: forward query unchanged MongoDB Primary-->>App: results

3. Components

Oplog Sync

Tails local.oplog.rs with tailable-await cursor. Position saved AFTER flush. Failed batches retained for retry.

Query Translator (AST)

graph LR BSON[BSON] -->|parse| TREE[ExprNode Tree] -->|emit| SQL[SQL]

Supports: find(), aggregate($match, $group, $sort, $limit, $project, $count), operators $gt/$lt/$in/$or/$regex/$exists.

Cluster Support

Crash Recovery

stateDiagram-v2 [*] --> Tailing Tailing --> Flushing: batch full / interval Flushing --> Tailing: OK → save position Flushing --> Tailing: FAIL → keep rows Tailing --> [*]: SIGTERM → flush + save
Position saved only after successful flush. Crash before save → replay. ReplacingMergeTree deduplicates.

5. Configuration

mongo:
  uri: "mongodb://mongo-primary:27017,mongo-secondary1:27017,mongo-secondary2:27017/?replicaSet=rs0"
  database: "myapp"

clickhouse:
  host: "localhost"
  port: 8123
  database: "analytics"
  cluster: ""              # empty = standalone

sync:
  mode: "oplog"            # or "changestream"
  batch_size: 1000
  flush_interval_ms: 500

api:
  port: 9090

Validated at startup: required fields, port ranges, batch_size (1-1M), mode must be oplog or changestream.

6. Deployment

The default deployment uses a 3-node MongoDB replica set (rs0) with 1 primary and 2 secondaries. MongoFlux connects using the full replica set URI and automatically follows primary elections.
